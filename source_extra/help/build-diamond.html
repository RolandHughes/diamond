Compile & Link

Before building Diamond you will need to add a environment variable called CS_HOME. Ensure you have set this variable to the export path where the CopperSpice libraries were installed.

To compile and link Diamond a configure script must be generated by Autotools. This step is only required if you check out the Diamond source from our repository. The command for this is shown in Step 1 below. If you download the source the configure script is included.

Note: All of the Autotool commands are run from a Bash prompt. For information about installing a Bourne shell (Bash) on Windows, refer to Building for Windows.

(1)  Run Autoconf from your project directory with the syntax shown below. This step is run once or anytime configure.ac is modified. When this command runs successfully, there is no output displayed.

autoreconf -i


(2)  Run configure from your project directory with the syntax shown below. This step is run only one time or anytime the configure.ac is changed.

./configure


(3)  To build Diamond from your project directory use the syntax shown below. This step is run every time the Diamond source is modified.

make

To build Diamond and deploy to the location specified in your bin/Makefile.am, use the following syntax. Using the provided build files the deployment directory will be yourProjectFolder/deploy.

make deploy

To build Diamond and install to the prefix specified when configure was run, use the following syntax. Using the provided build files the install directory on X11 and OS X is /usr/local. On Windows the install directory will be yourProjectFolder/install.

make install

This following command is run from your project directory. It will delete all of the object and intermediate files in the 'bin' folder. The next time you run 'make' every file will be compiled, since they were deleted.

make clean
